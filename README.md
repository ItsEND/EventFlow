# EventFlow

Учебный проект ASP.NET Core Web API для управления мероприятиями с CRUD-операциями, фильтрацией, пагинацией, глобальной обработкой ошибок и unit-тестами.

## Что реализовано

- CRUD-операции для событий
- хранение данных в памяти приложения
- бизнес-логика вынесена в сервис
- сервис подключён через Dependency Injection
- валидация входных данных
- глобальная обработка исключений через middleware
- единый формат ошибок через ProblemDetails
- фильтрация событий по названию и диапазону дат
- пагинация списка событий
- Swagger для тестирования API
- unit-тесты для `EventService`

## Стек

- C#
- .NET 10
- ASP.NET Core Web API
- Swagger / Swashbuckle
- xUnit

## Структура сущности Event

Событие содержит поля:

- `Id` — идентификатор события (`Guid`)
- `Title` — название события
- `Description` — описание события
- `StartAt` — дата и время начала
- `EndAt` — дата и время окончания

## Валидация

Проверяется:

- обязательность `Title`
- максимальная длина `Description`
- обязательность `StartAt`
- обязательность `EndAt`
- `EndAt` должно быть позже `StartAt`
- `page` не может быть меньше `1`
- `pageSize` не может быть меньше `1`
- `pageSize` не может быть больше `50`

## Фильтрация и пагинация

Эндпоинт `GET /events` поддерживает query-параметры:

- `title` — поиск по названию, регистронезависимый, частичное совпадение
- `from` — вернуть события, начинающиеся не раньше указанной даты
- `to` — вернуть события, заканчивающиеся не позже указанной даты
- `page` — номер страницы, по умолчанию `1`
- `pageSize` — количество элементов на странице, по умолчанию `10`

Пример:

```http
GET /events?title=митап&from=2026-05-01&to=2026-05-31&page=1&pageSize=5
```

## Эндпоинты API

### Получить список событий

```http
GET /events
```

С поддержкой фильтрации и пагинации:

```http
GET /events?title=митап&from=2026-05-01&to=2026-05-31&page=1&pageSize=5
```

### Пример успешного ответа списка с пагинацией

```JSON
{
  "items": [
    {
      "id": "2f2f8d3e-4ef5-4f8e-bd95-2fcd11a54c6b",
      "title": "Митап по ASP.NET Core",
      "description": "Middleware, маршрутизация и обработка ошибок",
      "startAt": "2026-05-05T18:00:00",
      "endAt": "2026-05-05T20:00:00"
    }
  ],
  "currentPage": 1,
  "pageSize": 10,
  "totalPages": 1,
  "totalItems": 1,
  "totalItemsOnPage": 1
}
```

### Получить событие по id

```http
GET /events/{id}
```

### Создать событие

```http
POST /events
Content-Type: application/json
```

Пример тела запроса:

```JSON
{
  "title": "Team Meeting",
  "description": "Sprint planning",
  "startAt": "2026-04-01T10:00:00",
  "endAt": "2026-04-01T11:00:00"
}
```

### Обновить событие

```http
PUT /events/{id}
Content-Type: application/json
```

Пример тела запроса:

```JSON
{
  "title": "Updated Meeting",
  "description": "Updated description",
  "startAt": "2026-04-01T12:00:00",
  "endAt": "2026-04-01T13:00:00"
}
```

### Удалить событие

```http
DELETE /events/{id}
```

## Формат ошибок

Для всех ошибок используется единый JSON-формат `ProblemDetails`.

### Пример ошибки валидации
```JSON
{
  "type": "https://httpstatuses.com/400",
  "title": "Некорректный запрос",
  "status": 400,
  "detail": "Одна или несколько ошибок валидации",
  "instance": "/events"
}
```
### Пример ошибки, если ресурс не найден
```JSON
{
  "type": "https://httpstatuses.com/404",
  "title": "Ресурс не найден",
  "status": 404,
  "detail": "Event с ключом 2f2f8d3e-4ef5-4f8e-bd95-2fcd11a54c6b не найден.",
  "instance": "/events/2f2f8d3e-4ef5-4f8e-bd95-2fcd11a54c6b"
}
```
### Пример внутренней ошибки сервера
```JSON
{
  "type": "https://httpstatuses.com/500",
  "title": "Внутренняя ошибка сервера",
  "status": 500,
  "detail": "Произошла непредвиденная ошибка",
  "instance": "/events"
}
```


## HTTP-статусы

- `200 OK` - успешное получение или обновление
- `201 Created` - успешное создание
- `204 No Content` - успешное удаление
- `400 Bad Request` - ошибка валидации
- `404 Not Found` - событие не найдено
- `500 Internal Server Error` - непредвиденная ошибка сервера

## Особенности реализации

Данные хранятся в памяти приложения (`List<Event>`), поэтому после перезапуска приложения все созданные события удаляются.
